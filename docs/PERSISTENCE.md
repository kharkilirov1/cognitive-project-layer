# Persistence

CPL has two local persistence layers under `.cpl/`.

## Structural index: `.cpl/index.sqlite`

`index.sqlite` stores the deterministic project cognition that does not require
an embedding provider:

- files with size, modified time, SHA-256, language, source/config/entry flags;
- symbols;
- references/usages;
- rich chunks and stable chunk IDs;
- graph nodes and edges;
- build metadata and schema version.

Build it:

```bash
cpl index-build --root .
```

Refresh it after source/config changes:

```bash
cpl index-refresh --root .
cpl index-refresh --root . --max-incremental-files 256
```

Inspect it:

```bash
cpl index-db --root .
cpl index-freshness --root .
cpl index-db --root . --json
```

MCP tools:

```text
cpl_index_build
cpl_index_db
cpl_index_freshness
cpl_index_refresh
```

HTTP endpoints:

```text
POST /index/rebuild
POST /index/refresh
GET  /index-db
GET  /index/freshness
```

When the index is fresh, `CognitiveProjectLayer::initialize()` can warm-start
symbols, references, graph, and chunks from SQLite. If any indexed file is
changed, missing, extra, or on an unsupported schema, CPL falls back to the full
scanner/parser path instead of using stale context.

`index-refresh` tries a bounded changed-file refresh first. If the SQLite index
is missing, incompatible, corrupt, or too many files changed, CPL rebuilds the
structural index from the full project layer. The MCP stdio server runs this
same refresh path automatically before serving layer-backed tools unless
`CPL_INDEX_AUTO_REFRESH=0` is set. Tune the incremental cutoff with
`CPL_INDEX_REFRESH_LIMIT` or the `max_incremental_files` tool argument. The
MCP auto-refresh path is throttled by `CPL_INDEX_AUTO_REFRESH_INTERVAL_MS`
(default: `2000`) so warm repeated tool calls stay fast.

## Embedding vector DB: `.cpl/vector_db.json`

`vector_db.json` remains the compatibility format for semantic embedding search.
It stores chunk metadata plus dense vectors generated by one of the embedding
backends.

Build offline local-hash embeddings:

```bash
cpl embed-index --root . --backend local-hash --dimensions 1536
```

Build local neural embeddings with Ollama:

```bash
ollama pull nomic-embed-text
cpl embed-index --root . --backend ollama --model nomic-embed-text --dimensions 768
```

Inspect/search:

```bash
cpl vector-db --root .
cpl embed-search --root . "auth login route" --limit 10
```

## Git hygiene

Do not commit local cognition state:

```gitignore
.cpl/
```

`cpl init` writes that ignore rule automatically for new project configs.
